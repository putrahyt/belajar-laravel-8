#Tutorial 4 - Model, Collection, Controller
1. php artisan make:model NamaModel -> untuk buat kelas model

   php artisan make:model NamaModel --migation -> untuk migrasi menjadi tabel didalam database

2. CTRL + Shift + P untuk membuat perintah laravel di vs code jadi cuma ketikkan artisan trus pilih mau buat apa

3. Kalo manggil method static tinggal buat ::
   jadi buat dulu nama class nya trus tambahin ::

4. Kalau property biasa yang tidak ada kata static nya cara manggilnya yaitu tambahin $this->namaproperty;
   
   tapi kalau property nya static cara manggilnya tambahin self::namaproperty;

   contoh:
   class hewan {
     private static $jumlahkaki=4;
     private $warnahewan=hitam;

     public function kaki() {
        // static
        return self::$jumlahkaki;
     }

     public function warna() {
        // no static
        return $this->warnahewan;
     }
   }

5. Collection pembungkus sebuah array yang lebih sakti, memungkinkan menjalankan banyak function, cara buatnya tinggal tambahin collect(), lebih jauh lagi liat di dokumentasi laravel

   Before
   public static function all()
    {
        return self::$blog_posts; => $blog_posts ini adalah array
    }

   After
   public static function all()
    {
        return collect(self::$blog_posts);
    }

6. penggunaan Collection

            $post = $p;
        }
    }

    ini sesudah pakai Collection, cukup tambahin firstWhere
    return $posts->firstWhere('slug', $slug);


7. self:: itu untuk manggil property static sedangkan static:: itu untuk manggil method static

   contoh
   class hewan {
     private static $jumlahkaki=4;

     public static function kaki() {
        // static
        return self::$jumlahkaki; => manggil property static
     }

     public function hitungkaki() {
        return static::kaki(); => manggil method static
     }
   }

#Tutorial 7 - Post Category & Eloquent
1. Agar tabel nya saling terhubung maka di migration nya ditambah foreignId...contoh $table->foreignId('category_id');

2. Lalu tambahkan di modelnya relationship..misal  one to many, one to one, many to many
  
   Kalo ke kasus web kita, 1 Category bisa memiliki banyak post maka kardinalitasnya one to many, dan kebalikannya 1 post hanya punya 1 Category maka kardinalitasnya one to one

   jadi kalo one to many maka di Eloquentnya "hasMany"
   kalo one to one maka dibuat "belongsTo"

   ingat!! kasus kita relationship utamanya itu di tabel "kategori".
   Kalo hubungan kategori dan post itu one to one maka Eloquentnya "hasOne" dan kalo misalnya hubungan post ke kategorinya itu one to many maka Eloquentnya "belongsToMany"

   penggunaan belongsTo atau belongsToMany itu kalau tabelnya itu punya foreign key dari tabel utamanya

   nahh penggunaan nama foreign key nya itu defaultnya versi singular dari nama tabel dan ditambah (_id).. contoh nama tabelnya categories maka foreign key nya category_id.. contoh lagi nama tabelnya users maka foreign key nya user_id... bisa juga dicustom untuk nama nya tapi harus dikasih tau dulu di modelnya

   kalo mau paham lagi liat dokumentasi

3. Kalo mau menghubungkannya, maka didalam class modelnya buat nama method yang sama dengan nama modelnya..seperti ini

    public function category()
    {
        return $this->belongsTo(Category::class);
    }

    jadi itu mau hubungkan post dengan category..makanya nama methodnya harus sama dengan nama modelnya(ehh ga harus sama sihh)..cara manggilnya $post->category...atau $post->category->name

    nahh kalo mau nama methodnya diubah seperti ini
    public function cat()
    {
        return $this->belongsTo(Category::class);
    }

    maka tambahin
    public function cat()
    {
        return $this->belongsTo(Category::class, 'category_id');
    }

    maka dikasih tau juga untuk foreign key nya..jadi nama methodnya cat tapi untuk foreign key nya category_id
