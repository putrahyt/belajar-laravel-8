#Tutorial 4 - Model, Collection, Controller
1. php artisan make:model NamaModel -> untuk buat kelas model

   php artisan make:model NamaModel --migation -> untuk migrasi menjadi tabel didalam database

2. CTRL + Shift + P untuk membuat perintah laravel di vs code jadi cuma ketikkan artisan trus pilih mau buat apa

3. Kalo manggil method static tinggal buat ::
   jadi buat dulu nama class nya trus tambahin ::

4. Kalau property biasa yang tidak ada kata static nya cara manggilnya yaitu tambahin $this->namaproperty;
   
   tapi kalau property nya static cara manggilnya tambahin self::namaproperty;

   contoh:
   class hewan {
     private static $jumlahkaki=4;
     private $warnahewan=hitam;

     public function kaki() {
        // static
        return self::$jumlahkaki;
     }

     public function warna() {
        // no static
        return $this->warnahewan;
     }
   }

5. Collection pembungkus sebuah array yang lebih sakti, memungkinkan menjalankan banyak function, cara buatnya tinggal tambahin collect(), lebih jauh lagi liat di dokumentasi laravel

   Before
   public static function all()
    {
        return self::$blog_posts; => $blog_posts ini adalah array
    }

   After
   public static function all()
    {
        return collect(self::$blog_posts);
    }

6. penggunaan Collection
    ini codingan sebelum menggunakan Collection
    $post = [];
    foreach($posts as $p) {
        if($p['slug'] === $slug) {
            $post = $p;
        }
    }

    ini sesudah pakai Collection, cukup tambahin firstWhere
    return $posts->firstWhere('slug', $slug);


7. self:: itu untuk manggil property static sedangkan static:: itu untuk manggil method static

   contoh
   class hewan {
     private static $jumlahkaki=4;

     public static function kaki() {
        // static
        return self::$jumlahkaki; => manggil property static
     }

     public function hitungkaki() {
        return static::kaki(); => manggil method static
     }
   }

8. Cara buat controller yaitu php artisan make:controller NamaController

9. Route::get('/blog', [PostController::class, 'index']); => ini sudah pakai controller

#Tutorial 5 - Database, Migration, Eloquent
1. Ganti di env untuk setting db
2. untuk buat migrasi => php artisan make:Migration create_nama_migrasi
3. php artisan migrate untuk eksekusi jadi tabel
4. tabel migation itu kalo udah pakai git gunanya untuk melacak perubahan yang dilakukan pada tabel di db
5. php artisan migrate:rollback => untuk hapus tabel di db
6. php artisan migrate:fresh => untuk rollback dan migrate sekaligus
7. Untuk cari tipe data apa aja yg bisa dibuat di migrate, cari di  dokumentasi laravelnya yaitu column types
8. tinker adalah aplikasi dalam laravel digunakan untuk berinteraksi dengan data pada aplikasi laravel, ketikkan "php artisan tinker"
9. fillable gunanya field yang bisa diisi di tabel
10. cara jalanin tinkernya, $user = new App\Models\User
    lalu cara isi fieldnya
    $user->name = 'Nama';
    $user->email = 'Email';
    $user->password = bcrypt('12345);
    trus kalo mau save tinggal ketik
    $user->save();

    kalo mau liat semua isinya ketikkan $user->all()
    method all() itu udah milik Eloquent yang bisa dipakai
    hasil yang dikembalikan method all() sudah bentuk Collection

    $user->findOrFail(20) untuk cek apakah ada data dan kalo gaada data tampilkan pesan

#Tutorial 6 - Post
1. php artisan make:model NamaModel => untuk buat model

   jika ditambah -m akan buat migrasi
   jika ditambah -f ditambah file factory
   jika ditambah -c ditambah file controller

2. Jika kita buat migrasi baru dan mau menjalankan dengan perintah php artisan migrate maka dia hanya menambahkan migrasi baru tersebut saja tanpa mengganggu migrasi yang lain

3. - Post::pluck('title') => buat cari di tabel yang nama fieldnya title
   - Post::find(2) => buat cari di tabel yang index ke 2
   - Post::where('title', 'Judul Berubah') => buat cari data dalam tabel Post yang nama fieldnya title dan data nya "judul berubah"
   - Post::find(3)->update(['title' => 'Judul Ketiga Berubah]); => ini untuk update, yang artinya cari dulu index ke tiga dari tabel Post dan update title nya

4. {{ $post['id'] }} => ini masih pakai notasi array, bisa juga pakai notasi objek caranya {{ $post->id }}

5. kalau menggunakan {{  }} => artinya kita menjalankan php echo dan juga htmlspecialchars(kalau ada tag html maka akan di escape)..kalau mau tag html juga dieksekusi maka pakai {!!  !!}

6. ada cara lain dalam input data pakai tinke yaitu
   Post::create([
     'title' => 'Judul',
     'body' => 'Body'
   ])

   tetapi harus buat dulu di model Post nya yaitu protected $fillable = ['title', 'body']; agar bisa diisi

   ada perbedaan antara fillable dan guarded, kalo fillable itu yang boleh diisi dan sisanya gaboleh diisi sedangkan guarded yang gaboleh diisi dan sisanya boleh diisi

   tidak hanya create saja, update pun juga harus cek fillable atau guarded

7. Route model binding agar bisa kirim data tidak hanya id saja, bisa   yang lainnya contohnya mungkin title dsb

   route::get('post/{post}', function(Post $post) {
        return view('post');
   })

   function(Post $post) => Post itu model nya dan nama nya $post, harus sama di bagian get('post/{post}')

   kalau gunakan controller
   Route::get('posts/{post}', [PostController::class, 'show']);

   public function show(Post $post)
    {
        return view('post', [
            'title' => 'Single Post',
            // 'post' => Post::find($slug)
            'post' => $post
        ]);
    }

   karena ini hanya pakai get('post/{post}') => {post} maka itu identifiernya id, jika hanya ingin mencari slug maka tinggal tambahin {post:slug}

   intinya {post} harus sama pada bagian function(Post $post)
